## 1. 基础理论 (fundamentals)

```
你是一位精通机器学习、深度学习和数学基础的资深 AI 研究员。请为我系统、深入地讲解 AI 的基础理论体系，要求覆盖以下所有方向，每个方向都需要从原理到直觉理解进行阐述：

### 一、机器学习基础
1. 监督学习、无监督学习、强化学习的本质区别与联系，各自适用的问题范式
2. 偏差-方差权衡（Bias-Variance Tradeoff）的数学推导与直觉解释，过拟合/欠拟合的诊断方法
3. 损失函数设计哲学：交叉熵、MSE、Hinge Loss 等的数学意义与选择依据
4. 正则化技术（L1/L2/Dropout/Early Stopping）的几何直觉与为什么有效
5. 经典模型（SVM、决策树、随机森林、XGBoost）的核心思想与数学推导

### 二、深度学习基础
1. 神经网络的万能近似定理：为什么深度网络能拟合任意函数？宽度 vs 深度的权衡
2. 反向传播算法的完整数学推导（从链式法则到计算图）
3. 梯度消失/爆炸问题的根因分析与解决方案演进（ReLU → ResNet → Normalization）
4. 优化器演进路线：SGD → Momentum → Adam → AdamW，每一步改进解决了什么问题
5. CNN 的局部连接与权值共享为什么对视觉任务有效？从傅里叶变换视角理解卷积
6. RNN/LSTM/GRU 的记忆机制设计：门控机制如何解决长程依赖？

### 三、Transformer 架构（重点）
1. Self-Attention 的完整数学推导：Q/K/V 的几何意义，为什么要除以 √d_k
2. Multi-Head Attention 的动机：多头分别关注不同子空间的直觉理解
3. Positional Encoding 为什么用正弦/余弦？RoPE 等后续改进的原理
4. Transformer 为什么能取代 RNN？并行计算优势的本质分析
5. Layer Normalization vs Batch Normalization 在 Transformer 中的选择原因

### 四、概率统计与数学基础
1. 贝叶斯定理在机器学习中的核心地位：先验、似然、后验的直觉
2. 最大似然估计（MLE）vs 最大后验估计（MAP）的联系与区别
3. 信息论基础：熵、交叉熵、KL 散度在 ML 中的应用
4. 概率图模型基础：贝叶斯网络与马尔可夫随机场

### 五、认知科学视角
1. 人类认知与深度学习的类比与差异
2. 系统1/系统2思维与当前 AI 架构的关系
3. 认知科学如何启发下一代 AI 架构设计

请在讲解中注重：
- 用直觉和类比帮助理解抽象概念
- 关键公式的推导过程，而非仅列出结论
- 不同概念之间的联系与演进脉络
- 面试和实际工作中最常考察的知识点标注
```

## 2. 大语言模型 (llm)

```
你是一位在大语言模型领域有丰富研发经验的高级研究员，曾深度参与模型预训练、微调和推理部署。请为我系统深入地讲解大语言模型的完整技术栈，覆盖以下所有方面：

### 一、Transformer 架构演进
1. 从 Encoder-Only (BERT) → Decoder-Only (GPT) → Encoder-Decoder (T5) 的架构选择：为什么 GPT 路线最终胜出？
2. 现代 LLM 的关键架构改进：GQA (Grouped Query Attention)、SwiGLU 激活函数、RMSNorm、RoPE 的原理与效果
3. MoE (Mixture of Experts) 架构：稀疏激活如何在控制计算成本的同时扩大模型容量？负载均衡问题
4. 长上下文技术：从 2K → 128K → 1M 的窗口扩展技术路线（ALiBi、YaRN、Ring Attention）

### 二、预训练
1. 预训练数据工程：数据采集、清洗、去重、毒性过滤、质量评分的完整流水线
2. 预训练目标：Causal LM vs Masked LM vs Prefix LM 的区别和选择
3. Scaling Laws：Chinchilla 定律的含义，如何根据计算预算确定最优模型大小和数据量
4. 分布式训练：数据并行 (DDP/FSDP)、张量并行 (TP)、流水线并行 (PP) 的原理与适用场景
5. 训练稳定性：Loss Spike 的原因与处理，学习率调度策略

### 三、微调技术（重点）
1. **SFT (Supervised Fine-Tuning)**：指令微调的数据构造方法，对话格式设计，数据质量 vs 数量的权衡
2. **RLHF**：完整流程拆解——奖励模型训练 → PPO 训练，为什么 RLHF 能让模型"对齐"人类偏好？
3. **DPO (Direct Preference Optimization)**：如何绕过奖励模型直接优化？与 RLHF 的数学等价性证明
4. **LoRA/QLoRA**：低秩适配的数学原理，rank 的选择策略，QLoRA 的量化细节
5. 各种微调方法的对比：全参数微调 vs LoRA vs Adapter vs Prefix Tuning，如何选择？
6. 微调的常见坑：灾难性遗忘、过拟合小数据集、对话能力退化

### 四、推理与部署
1. 量化技术：INT8/INT4/GPTQ/AWQ/GGUF 的原理对比，精度损失分析
2. KV Cache 机制：为什么是推理加速的关键？PagedAttention (vLLM) 如何优化显存
3. 推理框架对比：vLLM vs TGI vs TensorRT-LLM vs SGLang 的架构差异与适用场景
4. Speculative Decoding（投机解码）：小模型辅助大模型加速的原理
5. 批处理优化：Continuous Batching 如何提高 GPU 利用率
6. 成本优化策略：模型选择、混合部署、缓存策略的工程实践

### 五、评估体系
1. 基础能力评估：MMLU、HumanEval、GSM8K 等 benchmark 的设计思路与局限性
2. 对齐评估：MT-Bench、AlpacaEval、Chatbot Arena 的方法论差异
3. 安全评估：对抗攻击鲁棒性、有害内容生成检测
4. 实际业务评估：如何设计面向特定场景的评估方案？

请在讲解中特别注意：
- 预训练和微调的数据工程最佳实践
- 不同模型规模（7B/13B/70B）的实际部署经验
- 技术选型的决策依据和 trade-off 分析
- 标注哪些知识点对面试最有价值
```

## 3. RAG 检索增强生成 (rag)

```
你是一位在 RAG (Retrieval-Augmented Generation) 系统方面有丰富生产经验的架构师，曾将 RAG 从 PoC 做到生产级。请为我系统深入地讲解 RAG 的完整技术体系，覆盖以下所有方面：

### 一、RAG 架构设计
1. Naive RAG → Advanced RAG → Modular RAG 的演进路线，每一代解决了什么问题
2. RAG 的核心数据流：Indexing → Retrieval → Generation 每个环节的关键设计决策
3. RAG vs Fine-Tuning vs Long Context：三种知识注入方式的适用场景决策树
4. 系统架构设计：离线索引流水线 + 在线检索链路的工程拆分

### 二、文档解析与分块
1. 不同文档格式的解析挑战：PDF（扫描版/原生版）、Word、HTML、Markdown 的处理策略
2. 表格和图片解析：版面分析（Layout Analysis）技术、OCR、多模态解析
3. 分块策略深度对比：
   - 固定大小分块 vs 语义分块 vs 递归分块
   - 基于文档结构（标题层级）的分块
   - 滑动窗口与 overlap 的最佳实践
4. 分块粒度对检索质量的影响：chunk_size 实验方法论
5. 元数据提取与富化：如何为每个 chunk 添加有价值的元数据

### 三、Embedding 模型
1. 文本嵌入模型的工作原理：从 Word2Vec → Sentence-BERT → E5/BGE 的演进
2. 对称检索 vs 非对称检索：query-doc vs query-query 场景的模型选择
3. 多语言嵌入、代码嵌入、跨模态嵌入的技术方案
4. Embedding 模型微调：如何用业务数据提升特定领域的检索效果
5. 维度选择与性能权衡：高维 vs 低维的 trade-off

### 四、向量数据库
1. 主流向量数据库对比：Milvus vs Weaviate vs Qdrant vs Pinecone vs pgvector
2. 索引类型深度解析：FLAT vs IVF vs HNSW vs DiskANN 的原理、适用场景和性能特征
3. 混合搜索：向量搜索 + 全文搜索（BM25）的融合策略
4. 大规模向量检索的工程挑战：分片、副本、内存管理
5. 过滤与元数据查询：pre-filter vs post-filter 的性能差异

### 五、检索策略（重点）
1. 基础检索增强：
   - HyDE (Hypothetical Document Embeddings)：生成假设文档再检索
   - Query Rewriting / Expansion：查询改写与扩展
   - Multi-Query：从多个角度检索
2. 高级检索模式：
   - Parent-Child Retrieval：小块检索、大块返回
   - Sentence Window Retrieval：句子级精确匹配 + 上下文窗口扩展
   - Recursive Retrieval：分层递进检索
3. 重排序（Reranking）：Cross-Encoder 重排序的原理与实践，Cohere Rerank vs BGE-Reranker
4. 自适应检索：让 LLM 决定是否需要检索、检索什么

### 六、生产级 RAG 十大挑战（重点）
逐一深入分析并给出解决方案：
1. 内容缺失 — 知识库覆盖不全
2. 错过排名靠前的文档 — 检索精度不够
3. 不在上下文中 — 检索到了但超出窗口
4. 未提取出来 — 模型未能利用上下文中的关键信息
5. 格式错误 — 输出格式不符合要求
6. 不正确的具体性 — 回答过于笼统或过于具体
7. 不完整 — 回答只覆盖了部分要点
8. 数据摄入可扩展性 — 海量文档的索引性能
9. 结构化数据处理 — 表格、数据库的 RAG
10. 多来源数据融合 — 不同数据源的一致性

### 七、RAG 评估
1. 评估维度：Faithfulness（忠实度）、Relevancy（相关性）、Context Recall/Precision
2. 评估框架：RAGAS、TruLens 的方法论对比
3. 构建评估数据集的方法：人工标注 vs LLM 辅助生成
4. 端到端评估 vs 模块级评估的策略

请在讲解中特别关注：
- 每个技术选择背后的工程权衡
- 从 PoC 到生产的常见踩坑经验
- 具体的性能数据和调优参数参考
- 用实际案例说明最佳实践
```

## 4. Prompt 工程 (prompt-engineering)

```
你是一位精通 Prompt Engineering 的资深 AI 应用工程师，对 LLM 的输入输出行为有深入理解。请为我系统深入地讲解 Prompt 工程的完整知识体系，要求覆盖从基础技巧到工程化管理的所有层面：

### 一、基础 Prompting 技术
1. 角色设定（Role Prompting）：为什么给 LLM 分配角色能提升输出质量？背后的注意力分配机制
2. 少样本学习（Few-Shot Learning）：示例选择策略、示例数量与质量的 trade-off、动态选择示例
3. 输出格式控制：JSON/XML/Markdown 等结构化输出的可靠方法，JSON Mode vs Function Calling
4. 分隔符与结构化输入：如何用标记降低指令注入风险并提升理解准确率
5. 温度与采样参数：temperature、top_p、top_k 对输出的精确影响，不同任务的最佳参数

### 二、高级推理技术（重点）
1. **Chain of Thought (CoT)**：
   - 为什么"让模型想一想"能提升推理能力？
   - Zero-Shot CoT vs Few-Shot CoT 的效果对比
   - CoT 的局限性：什么情况下 CoT 反而有害？
2. **Tree of Thought (ToT)**：多路径探索与回溯机制
3. **ReAct (Reasoning + Acting)**：思考-行动-观察循环的设计
4. **Self-Consistency**：多次采样取多数投票的原理与实现
5. **Reflection/Self-Critique**：让模型自我审查和修正

### 三、Prompt 设计模式
1. 元提示（Meta-Prompting）：让 LLM 生成 Prompt 的方法论
2. 链式提示（Prompt Chaining）：多步骤任务的分解与串联
3. 条件逻辑嵌入：在 Prompt 中实现 if-else 逻辑
4. 上下文压缩：长上下文场景下的信息压缩技巧
5. 幻觉控制模式：引用验证、置信度表达、拒绝回答的设计

### 四、System Prompt 设计
1. System Prompt 的作用域与优先级
2. 人格定义、能力边界、输出约束的层次化设计
3. 防御性设计：如何抵抗 Prompt Injection 和 Jailbreak
4. System Prompt 的版本管理与 A/B 测试方法论

### 五、Prompt 工程化管理
1. Prompt 作为代码：版本控制、模板引擎、变量管理
2. 业务逻辑与 Prompt 解耦：配置化管理方案
3. Prompt 评估框架：自动化回归测试、评分标准设计
4. Prompt 优化方法论：DSPy 等自动优化工具的原理与实践
5. 多模型适配：同一业务逻辑如何适配不同 LLM（GPT-4 vs Claude vs 开源模型）

### 六、安全与对抗
1. Prompt Injection 攻击分类：直接注入、间接注入、分隔符逃逸
2. Jailbreak 技术与防御：DAN、角色扮演攻击、多轮攻击
3. 数据泄露防护：防止模型输出训练数据或系统提示
4. 安全防护的多层架构：输入检测 → Prompt 防御 → 输出过滤

请在讲解中注重：
- 每种技术附带具体的 Prompt 模板示例
- 技术选择的决策流程图
- 常见失败模式与调试方法
- 与实际业务场景的结合
```

```
### 五、多 Agent 系统
1. 多 Agent 协作模式：
   - 层级式（Manager-Worker）
   - 对等式（Peer-to-Peer）
   - 辩论式（Debate）
2. Agent 间通信：消息传递、共享黑板、事件驱动
3. 任务分配与负载均衡
4. 冲突解决与共识机制
5. 框架对比：LangGraph vs CrewAI vs AutoGen vs MetaGPT

### 六、人机协作（HITL）
1. 何时需要人类介入：置信度阈值、高风险操作、模糊指令
2. 中断与恢复：如何设计可暂停/恢复的 Agent 工作流
3. 审批流程的集成
4. 反馈循环：从人类纠正中学习

### 七、Agent 工程化
1. 状态管理与持久化：检查点、会话管理

4. 错误处理：优雅降级、超时控制、死循环检测

请在讲解中特别关注：
```
