# 第一章：机器学习基础

## 1.1 学习范式总览

机器学习的三大范式——监督学习、无监督学习、强化学习——并非孤立存在，而是构成了一个连续的光谱。理解它们的本质区别和联系，是掌握整个 AI 技术栈的基础。

### 1.1.1 监督学习（Supervised Learning）

**本质定义**：给定输入-输出对的数据集 $\{(x_i, y_i)\}_{i=1}^{N}$，学习一个映射函数 $f: X \rightarrow Y$，使得 $f(x)$ 尽可能接近真实标签 $y$。

**核心思想**：从"标准答案"中学习规律。就像学生通过做有答案的练习题来学习解题方法。

**数学形式化**：

```
目标：min_θ L(θ) = (1/N) Σ_{i=1}^{N} ℓ(f_θ(x_i), y_i) + λR(θ)

其中：
- f_θ 是参数为 θ 的模型
- ℓ 是损失函数（衡量预测与真实的差距）
- R(θ) 是正则化项（防止过拟合）
- λ 是正则化系数
```

**两大子问题**：

| 问题类型 | 输出空间 | 典型损失函数 | 典型模型 | 评估指标 |
|---------|---------|------------|---------|---------|
| 分类 | 离散标签 {0,1,...,K} | 交叉熵损失 | LR, SVM, RF, XGBoost | 准确率, F1, AUC-ROC |
| 回归 | 连续实数 ℝ | 均方误差 MSE | 线性回归, 岭回归, SVR | MAE, RMSE, R² |

**适用问题范式**：
- 有明确的"正确答案"可供学习
- 标注数据充足（或可获得）
- 输入到输出存在可学习的映射关系

**面试重点** 🎯：监督学习的核心假设是训练数据和测试数据来自同一分布（i.i.d. 假设），当这个假设被违反时（distribution shift），模型性能会显著下降。

### 1.1.2 无监督学习（Unsupervised Learning）

**本质定义**：给定无标签数据集 $\{x_i\}_{i=1}^{N}$，发现数据中的内在结构和模式。

**核心思想**：没有"标准答案"，而是从数据本身发现规律。就像在没有地图的情况下探索一片未知领域，自行发现地形特征。

**主要任务**：

1. **聚类（Clustering）**：将数据分成有意义的组
   - K-Means：最小化组内距离之和
   - DBSCAN：基于密度的聚类，能发现任意形状的簇
   - 层次聚类：自底向上或自顶向下的合并/分裂

2. **降维（Dimensionality Reduction）**：在保留关键信息的前提下减少数据维度
   - PCA（主成分分析）：找到数据方差最大的方向
   - t-SNE：保持局部邻域结构的非线性降维
   - UMAP：兼顾全局和局部结构

3. **生成模型（Generative Models）**：学习数据的概率分布 $P(X)$
   - VAE（变分自编码器）：通过潜在空间重建数据
   - GAN（生成对抗网络）：生成器和判别器的对抗博弈
   - 扩散模型：通过去噪过程生成数据

**与监督学习的联系**：
- 无监督预训练 + 监督微调是现代 AI 的主流范式（如 BERT 的 MLM 预训练 → 下游任务微调）
- LLM 的预训练本质上是无监督学习（Next Token Prediction）

### 1.1.3 强化学习（Reinforcement Learning）⭐ 与 LLM 对齐直接相关

**本质定义**：智能体（Agent）在环境（Environment）中通过试错学习最优策略（Policy），以最大化累积奖励（Cumulative Reward）。

**核心思想**：通过"试错"和"延迟反馈"学习决策。就像训练宠物：做对了给奖励，做错了不给，宠物逐渐学会什么行为能获得奖励。

**数学框架——马尔可夫决策过程（MDP）**：

```
MDP = (S, A, P, R, γ)

- S：状态空间（所有可能的状态）
- A：动作空间（所有可能的动作）
- P(s'|s,a)：状态转移概率（采取动作 a 后从状态 s 转到 s' 的概率）
- R(s,a)：即时奖励函数
- γ ∈ [0,1]：折扣因子（未来奖励的衰减系数）

目标：找到最优策略 π* = argmax_π E[Σ_{t=0}^{∞} γ^t R(s_t, a_t)]
```

**与 LLM 的关键联系**：

RLHF（基于人类反馈的强化学习）直接将 RL 应用于 LLM 训练：
- **状态**：当前对话上下文
- **动作**：生成下一个 token
- **奖励**：人类偏好评分（通过奖励模型近似）
- **策略**：LLM 本身

PPO（Proximal Policy Optimization）是 RLHF 中最常用的算法，其核心思想是限制每次策略更新的幅度，避免训练不稳定。

**面试重点** 🎯：能清晰解释 RLHF 的完整流程（SFT → 奖励模型训练 → PPO 优化）是大模型方向面试的高频考点。

### 1.1.4 三大范式的对比与联系

| 维度 | 监督学习 | 无监督学习 | 强化学习 |
|------|---------|-----------|---------|
| 数据形式 | (x, y) 标注对 | x 无标签数据 | (s, a, r, s') 交互序列 |
| 反馈信号 | 即时、精确 | 无反馈 | 延迟、稀疏 |
| 学习目标 | 预测 y | 发现结构 | 最大化累积奖励 |
| 典型应用 | 分类、回归 | 聚类、生成 | 游戏、机器人、对齐 |
| 在 LLM 中 | SFT 微调 | 预训练 (Next Token) | RLHF/PPO |

**统一视角**：现代 LLM 的训练流程恰好是三大范式的有机结合：
1. 预训练（无监督）→ 学习语言的统计规律
2. SFT（监督）→ 学习遵循指令
3. RLHF（强化学习）→ 对齐人类偏好

## 1.2 偏差-方差权衡（Bias-Variance Tradeoff）

### 1.2.1 数学推导

对于一个学习算法，其在新数据点 $x$ 上的期望预测误差可以分解为：

```
E[(y - f̂(x))²] = Bias²(f̂(x)) + Var(f̂(x)) + σ²

其中：
- Bias²(f̂(x)) = [E[f̂(x)] - f(x)]²    → 偏差的平方
- Var(f̂(x)) = E[(f̂(x) - E[f̂(x)])²]    → 方差
- σ²                                      → 不可约误差（数据本身的噪声）
```

**推导过程**：

```
E[(y - f̂(x))²]
= E[(y - f(x) + f(x) - E[f̂(x)] + E[f̂(x)] - f̂(x))²]
= E[(y - f(x))²] + E[(f(x) - E[f̂(x)])²] + E[(E[f̂(x)] - f̂(x))²]
  + 交叉项（期望为0）
= σ² + Bias² + Var
```

### 1.2.2 直觉解释

用"射箭打靶"的比喻：

```
高偏差 + 低方差          低偏差 + 高方差          低偏差 + 低方差
（欠拟合）              （过拟合）              （理想状态）

    ·  ·                  ·                      ·
   ·  ·  ·            ·       ·               ·  ⊙  ·
    ·  ·               ·    ·                    ·
                            ·

箭头集中但偏离靶心      箭头分散但围绕靶心      箭头集中且在靶心
```

- **高偏差**：模型太简单，无法捕捉数据的真实模式（系统性偏差）
- **高方差**：模型太复杂，对训练数据的噪声过度敏感（不稳定）
- **不可约误差**：数据本身的噪声，任何模型都无法消除

### 1.2.3 过拟合/欠拟合的诊断方法

**诊断流程图**：

```
训练误差高？
  ├─ 是 → 欠拟合（高偏差）
  │       解决方案：增加模型复杂度、增加特征、减少正则化
  │
  └─ 否 → 验证误差也低？
           ├─ 是 → 理想状态 ✓
           └─ 否 → 过拟合（高方差）
                   解决方案：增加训练数据、增加正则化、减少模型复杂度
```

**学习曲线分析**：

```
误差
 ↑
 │  ╲  训练误差（随数据量上升趋于稳定）
 │   ╲_______________
 │                    → 两条线之间的 gap 反映过拟合程度
 │   _______________
 │  ╱  验证误差（随数据量下降趋于稳定）
 │╱
 └──────────────────→ 训练数据量
```

## 1.3 损失函数设计哲学

损失函数是机器学习的"灵魂"——它定义了模型的优化目标。选择错误的损失函数，即使模型再强大也无法得到好的结果。

### 1.3.1 交叉熵损失（Cross-Entropy Loss）

**数学定义**：

```
二分类：L = -[y·log(p) + (1-y)·log(1-p)]
多分类：L = -Σ_{c=1}^{C} y_c · log(p_c)
```

**为什么用交叉熵而不用 MSE 做分类？**

1. **梯度特性**：交叉熵的梯度 $\frac{\partial L}{\partial z} = p - y$ 不会因为 sigmoid 饱和而消失，而 MSE 的梯度 $\frac{\partial L}{\partial z} = (p-y) \cdot p \cdot (1-p)$ 在 sigmoid 饱和时梯度趋近于零。

2. **信息论解释**：交叉熵 $H(P, Q)$ 衡量的是用分布 $Q$ 编码分布 $P$ 的信息量。最小化交叉熵等价于最小化预测分布与真实分布之间的 KL 散度。

3. **概率解释**：最小化交叉熵等价于最大似然估计（MLE），这为分类问题提供了概率论的理论支撑。

**面试重点** 🎯：能解释交叉熵与 KL 散度、最大似然估计之间的等价关系。

### 1.3.2 均方误差（MSE）

```
L = (1/N) Σ_{i=1}^{N} (y_i - f(x_i))²
```

**特性**：
- 对大误差惩罚严重（平方放大效应）
- 假设误差服从高斯分布时，MSE 等价于 MLE
- 对异常值敏感

**与 MAE 的对比**：

| 特性 | MSE | MAE |
|------|-----|-----|
| 梯度 | 与误差成正比 | 恒定大小 |
| 对异常值 | 敏感（平方放大） | 鲁棒 |
| 在零点 | 光滑 | 不可微 |
| 概率假设 | 高斯噪声 | 拉普拉斯噪声 |

### 1.3.3 Hinge Loss

```
L = max(0, 1 - y · f(x))    其中 y ∈ {-1, +1}
```

**几何直觉**：只要样本被正确分类且距离决策边界超过 margin（1），就不再产生损失。这正是 SVM 的核心思想——最大化分类间隔。

**Hinge Loss vs 交叉熵**：
- Hinge Loss 只关心"分对了且有足够间隔"，不关心概率
- 交叉熵关心概率校准（calibration），输出的概率值有意义
- 需要概率输出 → 交叉熵；只需分类决策 → Hinge Loss

## 1.4 正则化技术

正则化是对抗过拟合的核心武器。每种正则化技术背后都有清晰的几何直觉。

### 1.4.1 L1 正则化（Lasso）

```
L_regularized = L_original + λ Σ|w_i|
```

**几何直觉**：L1 约束区域是一个菱形（在高维空间是超八面体）。最优解倾向于落在菱形的"角"上，而角上意味着某些维度的参数恰好为零。

```
   w2
    ↑
    |╲
    | ╲  L1 约束（菱形）
    |  ╲
────+───╲────→ w1
    |  ╱
    | ╱
    |╱
    
等高线与菱形的交点倾向于在坐标轴上
→ 产生稀疏解（特征选择效果）
```

**效果**：自动特征选择——不重要的特征权重变为零，模型更加可解释。

### 1.4.2 L2 正则化（Ridge / Weight Decay）

```
L_regularized = L_original + λ Σ w_i²
```

**几何直觉**：L2 约束区域是一个球体（在高维空间是超球面）。球体没有"角"，因此解不会精确为零，而是所有权重都被均匀缩小。

```
   w2
    ↑
    |  ╭─╮
    | │   │  L2 约束（圆形）
    |  ╰─╯
────+────────→ w1
    
等高线与圆形的交点通常不在坐标轴上
→ 所有权重缩小但不为零
```

**效果**：权重平滑化——防止个别特征权重过大，提高模型稳定性。

**L1 vs L2 选择指南**：
- 特征很多但只有少数有用 → L1（特征选择）
- 特征都有用但可能过拟合 → L2（权重衰减）
- 两者结合 → Elastic Net

### 1.4.3 Dropout

**机制**：训练时随机将一部分神经元的输出置为零（以概率 $p$），测试时乘以 $(1-p)$ 或使用 inverted dropout。

**为什么有效？**

1. **集成学习视角**：每次 dropout 都相当于训练了一个不同的子网络。最终模型相当于指数级数量子网络的集成。

2. **打破共适应**：防止神经元之间产生过度依赖。每个神经元被迫独立学习有用的特征。

3. **贝叶斯近似**：Dropout 训练可以被解释为对贝叶斯神经网络的近似推断（Gal & Ghahramani, 2016）。

```python
# Inverted Dropout 实现
def dropout_forward(x, p=0.5, training=True):
    if training:
        mask = (np.random.rand(*x.shape) > p) / (1 - p)
        return x * mask
    else:
        return x  # 测试时不做任何操作
```

### 1.4.4 Early Stopping

**机制**：监控验证集损失，当验证集损失不再下降（或开始上升）时停止训练。

```
损失
 ↑
 │╲
 │ ╲  训练损失（持续下降）
 │  ╲_____________
 │     ╱‾‾‾‾‾‾‾‾‾  验证损失（先降后升）
 │   ╱
 │  ╱
 │ ╱
 └──────────────────→ 训练轮次
        ↑
    最佳停止点
```

**等价关系**：Early Stopping 可以被证明等价于 L2 正则化（在二次损失函数的假设下）——训练越久，参数偏离初始值越远；提前停止相当于约束参数不要离初始值太远。

## 1.5 经典模型深入

### 1.5.1 支持向量机（SVM）

**核心思想**：找到一个超平面，使得两类样本之间的"间隔"最大化。

**数学推导**：

```
线性可分情况：

超平面：w·x + b = 0
分类规则：y = sign(w·x + b)

间隔 = 2/||w||

目标：max 2/||w||  等价于  min (1/2)||w||²
约束：y_i(w·x_i + b) ≥ 1, ∀i

→ 这是一个凸二次规划问题
→ 通过拉格朗日对偶和 KKT 条件求解
```

**核函数技巧（Kernel Trick）**：

当数据线性不可分时，将数据映射到高维空间：$\phi(x)$。

核函数 $K(x_i, x_j) = \phi(x_i) \cdot \phi(x_j)$ 可以在不显式计算高维映射的情况下计算内积。

常用核函数：
- 线性核：$K(x_i, x_j) = x_i \cdot x_j$
- 多项式核：$K(x_i, x_j) = (x_i \cdot x_j + c)^d$
- RBF 核（高斯核）：$K(x_i, x_j) = \exp(-\gamma ||x_i - x_j||^2)$
- Sigmoid 核：$K(x_i, x_j) = \tanh(\alpha x_i \cdot x_j + c)$

**面试重点** 🎯：SVM 的核心三要素——最大间隔、支持向量、核函数。软间隔 SVM 中的 C 参数控制误分类惩罚（C 越大越不容忍误分类，模型越复杂）。

### 1.5.2 决策树

**核心思想**：通过一系列 if-else 规则将特征空间递归地划分为子区域。

**分裂准则**：

1. **信息增益（ID3）**：选择使信息增益最大的特征
   ```
   信息增益 = H(D) - Σ (|D_v|/|D|) · H(D_v)
   H(D) = -Σ p_k · log₂(p_k)  （熵）
   ```

2. **信息增益率（C4.5）**：除以特征本身的熵，避免偏向多值特征
   ```
   增益率 = 信息增益 / H_A(D)
   ```

3. **Gini 指数（CART）**：
   ```
   Gini(D) = 1 - Σ p_k²
   ```

**决策树的优缺点**：
- ✅ 可解释性强（可以直接看到决策规则）
- ✅ 不需要特征归一化
- ✅ 能处理混合类型特征
- ❌ 容易过拟合（通过剪枝缓解）
- ❌ 不稳定（数据微小变化可能导致完全不同的树）

### 1.5.3 随机森林（Random Forest）

**核心思想**：训练多棵决策树，每棵树用随机子集的数据和特征，最终投票/平均。

**两层随机性**：
1. **Bagging**：每棵树用有放回采样（Bootstrap）得到的数据子集
2. **特征随机选择**：每次分裂时只考虑随机子集的特征

**为什么有效？**

```
单棵决策树：高方差、低偏差
随机森林 = 多棵树取平均
→ 方差降低为 1/n（如果树之间独立）
→ 实际因为树之间有相关性，降低为 ρσ² + (1-ρ)σ²/n
   其中 ρ 是树之间的相关系数
→ 特征随机选择就是为了降低 ρ
```

### 1.5.4 XGBoost

**核心思想**：Boosting 方法——每棵新树拟合前面所有树的残差（梯度方向），逐步提升整体性能。

**与 Random Forest 的关键区别**：

| 特性 | Random Forest | XGBoost |
|------|--------------|---------|
| 集成方式 | 并行（Bagging） | 串行（Boosting） |
| 每棵树学什么 | 独立学习完整目标 | 学习前序模型的残差 |
| 降低什么 | 主要降低方差 | 同时降低偏差和方差 |
| 过拟合风险 | 较低 | 需要仔细调参 |

**XGBoost 的关键创新**：
1. 二阶泰勒展开近似损失函数
2. 正则化的目标函数（树的复杂度惩罚）
3. 列采样（类似 RF 的特征随机选择）
4. 稀疏感知算法（自动处理缺失值）
5. 近似分位数算法（处理大规模数据的特征分裂）

```
XGBoost 目标函数：
Obj = Σ ℓ(y_i, ŷ_i) + Σ Ω(f_k)

其中 Ω(f) = γT + (1/2)λ||w||²
- T：叶子节点数
- w：叶子节点权重
- γ, λ：正则化超参数
```

**面试重点** 🎯：XGBoost vs LightGBM vs CatBoost 的核心差异——LightGBM 用 Histogram-based 方法和 GOSS 采样加速，CatBoost 对类别特征原生支持更好。

## 1.6 模型选择与评估

### 1.6.1 评估指标体系

**分类指标**：

```
混淆矩阵：
                预测正例    预测负例
实际正例         TP          FN
实际负例         FP          TN

准确率 = (TP+TN) / (TP+TN+FP+FN)
精确率 = TP / (TP+FP)         → 预测为正的里面有多少真正是正的
召回率 = TP / (TP+FN)         → 所有真正为正的里面找回了多少
F1    = 2 × P × R / (P + R)  → 精确率和召回率的调和平均
```

**AUC-ROC**：
- ROC 曲线：以 FPR 为横轴、TPR 为纵轴
- AUC：ROC 曲线下面积，衡量模型在所有阈值下的综合表现
- AUC = 0.5 → 随机猜测
- AUC = 1.0 → 完美分类
- AUC 的概率解释：随机选一个正样本和一个负样本，模型给正样本的分数高于负样本的概率

**何时用什么指标**：
- 类别平衡 → 准确率
- 类别不平衡 → F1 或 AUC
- 误报代价高（如垃圾邮件过滤）→ 高精确率
- 漏报代价高（如疾病筛查）→ 高召回率

### 1.6.2 交叉验证

```
K-Fold 交叉验证：

数据集：[1] [2] [3] [4] [5]

Fold 1:  测试 [1]  训练 [2][3][4][5]
Fold 2:  测试 [2]  训练 [1][3][4][5]
Fold 3:  测试 [3]  训练 [1][2][4][5]
Fold 4:  测试 [4]  训练 [1][2][3][5]
Fold 5:  测试 [5]  训练 [1][2][3][4]

最终评分 = 5 次评分的平均值 ± 标准差
```

**变体**：
- 分层 K-Fold：保持每个 fold 中类别比例与整体一致
- 留一法（LOO）：K = N，计算成本高但偏差最小
- 时间序列交叉验证：只用过去的数据训练，未来的数据测试（避免数据泄露）

### 1.6.3 超参数调优

| 方法 | 原理 | 优点 | 缺点 |
|------|------|------|------|
| 网格搜索 | 穷举所有组合 | 简单可靠 | 计算量指数增长 |
| 随机搜索 | 随机采样组合 | 高维时效率高 | 可能错过最优 |
| 贝叶斯优化 | 用高斯过程建模目标函数 | 样本效率高 | 实现复杂 |
| Hyperband | 早停 + 资源自适应分配 | 快速 | 假设单调性 |

**面试重点** 🎯：随机搜索在高维超参数空间中通常比网格搜索更有效，因为不同超参数的重要性通常不同——随机搜索能在重要维度上覆盖更多值。

# 第二章：深度学习基础

## 2.1 神经网络的万能近似定理

### 2.1.1 定理陈述

**万能近似定理（Universal Approximation Theorem）**：一个具有单个隐藏层的前馈神经网络，只要有足够多的神经元，可以以任意精度逼近任何连续函数。

**形式化**：对于任何连续函数 $f: [0,1]^n \rightarrow \mathbb{R}$ 和任何 $\epsilon > 0$，存在一个前馈网络 $g(x)$，使得 $|f(x) - g(x)| < \epsilon$ 对所有 $x \in [0,1]^n$ 成立。

### 2.1.2 直觉理解

想象用乐高积木搭建任何形状：
- 每个积木块（神经元）可以表示一个简单的"阶梯函数"
- 足够多的积木块组合在一起，可以逼近任何复杂形状
- 每个 ReLU 神经元 $\max(0, wx+b)$ 是一个"折线段"
- 多个折线段组合 = 分段线性函数 → 逼近任何连续函数

```
单个 ReLU：           多个 ReLU 组合：        逼近目标函数：
    ╱                   ╱╲    ╱╲              ∿∿∿∿∿
   ╱                  ╱    ╲╱    ╲
──╱──              ──╱────────────╲──        ≈ ╱╲╱╲╱╲
```

### 2.1.3 宽度 vs 深度的权衡

**宽度（Wide Networks）**：
- 理论上单隐层就够（万能近似定理）
- 但可能需要指数级的神经元数量
- 参数效率低

**深度（Deep Networks）**：
- 深层网络可以用更少的参数表示更复杂的函数
- 每一层学习不同层次的抽象
- 类比人类认知：像素 → 边缘 → 纹理 → 部件 → 物体

**关键洞察**：深度网络的表达能力是指数级优于等宽度的浅层网络。

```
示例：表示一个包含 n 个变量的对称布尔函数

宽网络（1层）：需要 O(2^n) 个神经元
深网络（O(n)层）：每层只需要 O(n) 个神经元

→ 深度的"指数级压缩"效应
```

**面试重点** 🎯：为什么现代模型越来越深而不是越来越宽？答案是深度能以更少参数捕获更复杂的层次化特征。但深度也带来了梯度消失/爆炸问题，这是 ResNet、Normalization 等技术存在的原因。

## 2.2 反向传播算法（Backpropagation）

### 2.2.1 前向传播

```
输入 x → 第1层 → 第2层 → ... → 第L层 → 输出 ŷ → 损失 L

每一层的计算：
  z^(l) = W^(l) · a^(l-1) + b^(l)    （线性变换）
  a^(l) = σ(z^(l))                     （激活函数）

其中 a^(0) = x（输入），a^(L) = ŷ（输出）
```

### 2.2.2 链式法则推导

反向传播的核心是链式法则（Chain Rule）：

```
目标：计算 ∂L/∂W^(l) 和 ∂L/∂b^(l)（每一层的参数梯度）

定义 δ^(l) = ∂L/∂z^(l)（第 l 层的"误差信号"）

从输出层开始：
  δ^(L) = ∂L/∂ŷ · σ'(z^(L))

逐层反传：
  δ^(l) = (W^(l+1))^T · δ^(l+1) · σ'(z^(l))

参数梯度：
  ∂L/∂W^(l) = δ^(l) · (a^(l-1))^T
  ∂L/∂b^(l) = δ^(l)
```

### 2.2.3 计算图视角

现代深度学习框架（PyTorch、TensorFlow）使用计算图（Computational Graph）自动计算梯度。

```
计算图示例：L = (wx + b - y)²

       L
       |
      (·)²
       |
      (-)
     /   \
   (+)    y
   / \
 (*)   b
 / \
w   x

前向传播：从叶节点到根节点计算每个节点的值
反向传播：从根节点到叶节点计算每个节点的梯度

∂L/∂w = ∂L/∂(wx+b-y) · ∂(wx+b-y)/∂(wx) · ∂(wx)/∂w
       = 2(wx+b-y) · 1 · x
```

**PyTorch 自动微分示例**：

```python
import torch

# 定义参数
w = torch.tensor(2.0, requires_grad=True)
b = torch.tensor(1.0, requires_grad=True)
x = torch.tensor(3.0)
y = torch.tensor(7.0)

# 前向传播
y_pred = w * x + b
loss = (y_pred - y) ** 2

# 反向传播（自动计算所有梯度）
loss.backward()

print(f"∂L/∂w = {w.grad}")  # 2 * (2*3+1-7) * 3 = 2 * 0 * 3 = 0
print(f"∂L/∂b = {b.grad}")  # 2 * (2*3+1-7) * 1 = 0
```

## 2.3 梯度消失/爆炸问题

### 2.3.1 根因分析

```
考虑 L 层网络的梯度传播：

∂L/∂W^(1) = ∂L/∂z^(L) · ∂z^(L)/∂z^(L-1) · ... · ∂z^(2)/∂z^(1) · ∂z^(1)/∂W^(1)

= δ^(L) · Π_{l=1}^{L-1} [W^(l+1) · diag(σ'(z^(l)))]  · a^(0)

关键观察：梯度是 L-1 个矩阵的连乘积
```

**梯度消失**：如果每个乘积项的范数 < 1，则梯度指数级衰减
- Sigmoid 的导数最大值为 0.25 → 连续乘 L 次 → 趋近于零
- 结果：前面的层几乎无法更新，学不到有用的特征

**梯度爆炸**：如果每个乘积项的范数 > 1，则梯度指数级增长
- 权重初始化不当时容易发生
- 结果：参数更新过大，训练不稳定

### 2.3.2 解决方案演进

**第一阶段：ReLU 激活函数**

```
Sigmoid: σ(x) = 1/(1+e^(-x))      → σ'(x) ∈ (0, 0.25]  → 梯度消失
Tanh:    σ(x) = (e^x-e^(-x))/(e^x+e^(-x)) → σ'(x) ∈ (0, 1]
ReLU:    σ(x) = max(0, x)          → σ'(x) ∈ {0, 1}     → 正区间梯度恒为1

ReLU 变体：
- Leaky ReLU: max(αx, x), α=0.01    → 解决"死神经元"问题
- GELU:      x · Φ(x)               → GPT/BERT 使用
- SiLU/Swish: x · σ(x)              → 平滑版 ReLU
- SwiGLU:    门控 + SiLU             → LLaMA 等现代 LLM 使用
```

**第二阶段：残差连接（ResNet, 2015）**

```
普通网络：y = F(x)
残差网络：y = F(x) + x    ← 恒等映射的捷径

梯度传播：
∂y/∂x = ∂F(x)/∂x + 1

因为 +1 的存在，即使 ∂F(x)/∂x 很小，梯度也不会消失！
→ 这就是为什么 ResNet 可以训练 100+ 层的网络
```

**直觉理解**：残差连接让网络学习"需要改变什么"（残差），而不是"输出应该是什么"。这更容易——当最优映射接近恒等映射时，学习零映射比学习恒等映射容易得多。

**第三阶段：归一化技术**

**Batch Normalization (BN)**：

```
对每个特征在 mini-batch 上归一化：
  μ_B = (1/m) Σ x_i        （batch 均值）
  σ²_B = (1/m) Σ (x_i - μ_B)²  （batch 方差）
  x̂_i = (x_i - μ_B) / √(σ²_B + ε)  （归一化）
  y_i = γ · x̂_i + β        （可学习的缩放和偏移）
```

**Layer Normalization (LN)**：

```
对每个样本的所有特征归一化（而非跨 batch）：
  μ = (1/d) Σ_{j=1}^{d} x_j
  σ² = (1/d) Σ_{j=1}^{d} (x_j - μ)²
  x̂_j = (x_j - μ) / √(σ² + ε)
  y_j = γ · x̂_j + β
```

**BN vs LN 的选择**：

| 特性 | Batch Norm | Layer Norm |
|------|-----------|------------|
| 归一化维度 | 跨样本（batch 维度） | 跨特征（特征维度） |
| 依赖 batch size | 是（batch 太小效果差） | 否 |
| 适用架构 | CNN | Transformer / RNN |
| 训练/推理一致性 | 推理时用移动平均，有差异 | 完全一致 |

**Transformer 选择 LN 的原因**：
1. 序列长度可变，BN 在序列维度上不合适
2. 自回归生成时 batch size 通常为 1
3. LN 不依赖 batch 统计量，训练推理更一致

**RMSNorm（现代 LLM 使用）**：

```
简化版 Layer Norm，去掉了均值中心化，只做方差归一化：
  RMS(x) = √((1/d) Σ x_j²)
  x̂_j = x_j / RMS(x) · γ

优势：计算量减少约 15%，效果几乎无差
LLaMA, Qwen 等现代模型都使用 RMSNorm
```

## 2.4 优化器演进路线

### 2.4.1 SGD（随机梯度下降）

```
θ_{t+1} = θ_t - η · g_t

其中 g_t = ∇L(θ_t; x_i, y_i)  是单个样本（或 mini-batch）的梯度
```

**问题**：
- 梯度方向嘈杂（只用一个/少量样本估计梯度）
- 所有参数用相同的学习率
- 容易陷入鞍点（高维空间中鞍点远多于局部最小值）
- 在"狭窄山谷"中震荡（不同方向的曲率差异很大）

### 2.4.2 Momentum（动量）

```
v_{t+1} = β · v_t + g_t           （累积历史梯度方向）
θ_{t+1} = θ_t - η · v_{t+1}

β 通常取 0.9
```

**解决的问题**：减少震荡，加速在一致方向上的移动

**物理直觉**：想象一个球在曲面上滚动——球的速度不仅取决于当前位置的坡度，还受到之前运动方向的惯性影响。

```
无动量：                有动量：
  ↗ ↘ ↗ ↘             ────────→
  震荡前进              平滑加速
```

### 2.4.3 AdaGrad

```
G_t = G_{t-1} + g_t²              （累积历史梯度平方和）
θ_{t+1} = θ_t - η/√(G_t + ε) · g_t
```

**解决的问题**：自适应学习率——频繁更新的参数用小学习率，稀疏梯度的参数用大学习率

**问题**：$G_t$ 单调递增，学习率单调递减，训练后期学习率趋近于零，过早停止学习

### 2.4.4 RMSProp

```
v_t = β · v_{t-1} + (1-β) · g_t²    （指数移动平均代替累积和）
θ_{t+1} = θ_t - η/√(v_t + ε) · g_t

β 通常取 0.999
```

**改进**：用指数移动平均代替累积和，避免学习率无限衰减

### 2.4.5 Adam（Adaptive Moment Estimation）

```
m_t = β₁ · m_{t-1} + (1-β₁) · g_t        （一阶矩估计：梯度均值）
v_t = β₂ · v_{t-1} + (1-β₂) · g_t²       （二阶矩估计：梯度方差）

偏差校正：
m̂_t = m_t / (1 - β₁^t)
v̂_t = v_t / (1 - β₂^t)

参数更新：
θ_{t+1} = θ_t - η · m̂_t / (√v̂_t + ε)

默认超参数：β₁=0.9, β₂=0.999, ε=1e-8
```

**综合了 Momentum 和 RMSProp 的优点**：
- 动量 m_t：加速在一致方向上的移动（来自 Momentum）
- 自适应学习率 v_t：不同参数不同学习率（来自 RMSProp）
- 偏差校正：解决初始估计偏向零的问题

### 2.4.6 AdamW（Weight Decay 修正）

**Adam 的权重衰减问题**：在 Adam 中，L2 正则化和 weight decay 不等价！

```
Adam + L2 正则化：
  g_t = ∇L(θ_t) + λθ_t
  → L2 项的梯度也会被自适应学习率缩放 → 效果不一致

AdamW（解耦 weight decay）：
  θ_{t+1} = θ_t - η · (m̂_t/√v̂_t + λθ_t)
  → weight decay 直接作用于参数，不受自适应学习率影响
```

**AdamW 是当前 LLM 训练的标配优化器。**

### 2.4.7 优化器选择指南

```
决策流程：

需要训练 LLM / Transformer？
  └─ 是 → AdamW（几乎不需要纠结）
  
需要训练 CNN（计算机视觉）？
  └─ SGD + Momentum 通常泛化更好
  └─ AdamW 也可以，但可能需要更多正则化
  
稀疏特征 / NLP 嵌入？
  └─ Adam / AdamW（自适应学习率对稀疏特征更友好）
  
微调预训练模型？
  └─ AdamW + 小学习率（1e-5 ~ 5e-5）
```

## 2.5 CNN（卷积神经网络）

### 2.5.1 核心思想

**局部连接**：每个神经元只关注输入的一个局部区域（感受野），而非全连接。

**权值共享**：同一个卷积核在整个输入上滑动，所有位置共享同一组参数。

**为什么对视觉任务有效？**

1. **局部性**：图像中的语义特征通常是局部的（边缘、纹理、形状）
2. **平移不变性**：同样的特征可能出现在图像的任何位置
3. **参数效率**：共享权值大幅减少参数量

### 2.5.2 卷积操作的数学

```
2D 卷积：
  (f * g)(i, j) = Σ_m Σ_n f(m, n) · g(i-m, j-n)

实际实现（互相关）：
  output(i, j) = Σ_m Σ_n input(i+m, j+n) · kernel(m, n)

参数：
- 卷积核大小：通常 3×3 或 5×5
- 步长（stride）：滑动步长，stride > 1 实现下采样
- 填充（padding）：边缘填零以保持输出大小
- 通道数：输入通道 × 输出通道 × 核大小
```

### 2.5.3 从傅里叶变换视角理解卷积

```
卷积定理：在空间域的卷积 = 在频率域的逐元素乘法

f * g = F^(-1)[F(f) · F(g)]

直觉：
- 卷积核在频率域是一个滤波器
- 3×3 卷积核 → 高频滤波（边缘检测）
- 大卷积核 → 低频通过（模糊效果）
- CNN 学习的就是一组自适应的频率滤波器
```

### 2.5.4 经典 CNN 架构演进

```
LeNet (1998) → AlexNet (2012) → VGG (2014) → GoogLeNet (2014)
→ ResNet (2015) → DenseNet (2017) → EfficientNet (2019)
→ ConvNeXt (2022，用 Transformer 设计理念改造 CNN)
```

**ResNet 残差连接的深远影响**：
- 解决了深层网络的训练困难
- 成为后续几乎所有深度网络的标配
- Transformer 中也使用了残差连接

## 2.6 RNN / LSTM / GRU

### 2.6.1 RNN 基础

```
h_t = σ(W_hh · h_{t-1} + W_xh · x_t + b_h)
y_t = W_hy · h_t + b_y

h_t 是隐藏状态，携带了序列的"记忆"
```

**问题**：长程依赖问题——梯度通过时间步反传时，会指数级衰减/爆炸。

### 2.6.2 LSTM（长短期记忆网络）

**核心设计**：引入"细胞状态"（Cell State）作为信息高速公路，以及三个"门"来控制信息流。

```
遗忘门：f_t = σ(W_f · [h_{t-1}, x_t] + b_f)    → 决定丢弃哪些旧信息
输入门：i_t = σ(W_i · [h_{t-1}, x_t] + b_i)    → 决定存入哪些新信息
候选值：C̃_t = tanh(W_C · [h_{t-1}, x_t] + b_C)  → 新信息的内容
输出门：o_t = σ(W_o · [h_{t-1}, x_t] + b_o)    → 决定输出什么

细胞状态更新：C_t = f_t ⊙ C_{t-1} + i_t ⊙ C̃_t
隐藏状态更新：h_t = o_t ⊙ tanh(C_t)

⊙ 表示逐元素乘法
```

**为什么能解决长程依赖？**

细胞状态 $C_t$ 的更新路径是加法（而非乘法），梯度可以沿着细胞状态"无阻碍"地传播：
```
∂C_t/∂C_{t-1} = f_t  （遗忘门的值，接近1时梯度几乎不衰减）
```

### 2.6.3 GRU（门控循环单元）

LSTM 的简化版本，将遗忘门和输入门合并，减少参数量：

```
更新门：z_t = σ(W_z · [h_{t-1}, x_t])         → 类似 LSTM 的遗忘门
重置门：r_t = σ(W_r · [h_{t-1}, x_t])         → 控制使用多少历史信息
候选：  h̃_t = tanh(W · [r_t ⊙ h_{t-1}, x_t]) → 新的候选隐藏状态
输出：  h_t = (1-z_t) ⊙ h_{t-1} + z_t ⊙ h̃_t → 旧状态与新候选的加权平均
```

**LSTM vs GRU**：
- 参数：GRU 更少（2个门 vs 3个门）
- 性能：在大多数任务上效果相近
- 训练速度：GRU 更快
- 长序列：LSTM 理论上更好（独立的细胞状态）

**面试重点** 🎯：理解门控机制如何解决长程依赖，以及为什么 Transformer 最终取代了 RNN——并行计算能力和更强的长程依赖建模。

## 2.7 扩散模型（Diffusion Models）

### 2.7.1 核心思想

扩散模型是当前图像生成的主流方法（Stable Diffusion、DALL-E、Midjourney）。

```
前向过程（加噪）：逐步向数据添加高斯噪声，直到变成纯噪声
  x_0 → x_1 → x_2 → ... → x_T ≈ N(0, I)

反向过程（去噪）：学习从噪声中逐步恢复数据
  x_T → x_{T-1} → ... → x_1 → x_0

训练目标：学习每一步去噪的过程
  → 本质上是训练一个噪声预测网络 ε_θ(x_t, t)
```

### 2.7.2 数学框架

```
前向过程（有闭合解）：
  q(x_t | x_0) = N(x_t; √ᾱ_t · x_0, (1-ᾱ_t) · I)
  
  其中 ᾱ_t = Π_{s=1}^{t} (1 - β_s)，β_s 是噪声调度

→ 可以一步直接从 x_0 得到任意时间步的 x_t：
  x_t = √ᾱ_t · x_0 + √(1-ᾱ_t) · ε,  ε ~ N(0, I)

训练损失（简化版）：
  L = E_{t, x_0, ε} [||ε - ε_θ(x_t, t)||²]
  
  即：预测添加的噪声 vs 实际添加的噪声的 MSE
```

### 2.7.3 Latent Diffusion（Stable Diffusion 的核心）

```
直接在像素空间做扩散太慢（512×512×3 的图像）

Latent Diffusion 方案：
1. 用 VAE 编码器将图像压缩到潜在空间：z = E(x)
2. 在潜在空间做扩散（维度大幅降低）
3. 用 VAE 解码器将去噪结果恢复为图像：x = D(z)

优势：计算量减少 ~50 倍，效果相近
```

**面试重点** 🎯：扩散模型的核心训练目标可以用一句话概括——训练一个网络预测在不同噪声水平下添加到数据上的噪声。生成时从纯噪声开始，逐步去噪。

# 第三章：Transformer 架构（重点）

> Transformer 是现代 AI 的基石架构。从 BERT 到 GPT-4，从 ViT 到 Stable Diffusion，几乎所有前沿 AI 系统都建立在 Transformer 之上。深入理解 Transformer 的每个组件，是理解整个 LLM 技术栈的前提。

## 3.1 Self-Attention 的完整数学推导

### 3.1.1 从直觉出发

**核心问题**：给定一个序列（如一句话），序列中的每个位置如何"关注"其他位置来理解上下文？

**类比**：阅读理解时，你在读每个词的时候，会自动关注句子中与当前词最相关的其他词。比如读"猫坐在垫子上因为**它**很累"，读到"它"时你会关注"猫"。

### 3.1.2 Q/K/V 的定义与含义

给定输入序列 $X \in \mathbb{R}^{n \times d}$（n 个 token，每个 d 维）：

```
Q = X · W_Q    （Query: "我在找什么？"）
K = X · W_K    （Key: "我有什么信息？"）
V = X · W_V    （Value: "我的信息内容是什么？"）

其中 W_Q, W_K ∈ ℝ^{d × d_k}, W_V ∈ ℝ^{d × d_v}
```

**Q/K/V 的几何意义**：

- **Query（查询）**：当前位置的"提问"——"我需要什么样的上下文信息？"
- **Key（键）**：每个位置的"标签"——"我能提供什么样的信息？"
- **Value（值）**：每个位置的"内容"——"我实际的信息是什么？"

**注意力机制 = 信息检索**：
- 用 Query 去匹配所有的 Key
- 匹配度高的 Key 对应的 Value 获得更大权重
- 最终输出是 Value 的加权和

### 3.1.3 注意力计算

```
Attention(Q, K, V) = softmax(Q · K^T / √d_k) · V

步骤分解：
1. Q · K^T ∈ ℝ^{n × n}        → 注意力分数矩阵（每对位置的"相关度"）
2. / √d_k                      → 缩放（防止梯度消失）
3. softmax(·)                   → 归一化为概率分布
4. · V                          → 加权求和得到输出
```

**为什么要除以 √d_k？**

这是 Transformer 论文中一个精妙的设计。推导如下：

```
假设 q 和 k 的每个分量独立同分布，均值 0，方差 1
则 q · k = Σ_{i=1}^{d_k} q_i · k_i

E[q · k] = 0
Var[q · k] = d_k   （d_k 个方差为 1 的独立随机变量之和）

当 d_k 很大时，q · k 的值域很大
→ softmax 的输入值很大
→ softmax 输出趋向 one-hot（梯度接近零）
→ 训练困难

除以 √d_k 后：
Var[(q · k) / √d_k] = d_k / d_k = 1   → 方差标准化为 1
→ softmax 输入分布合理，梯度正常
```

**面试重点** 🎯：这个缩放因子是面试中最常被问到的 Transformer 细节之一。

### 3.1.4 注意力的矩阵视角

```
输入序列 X = [x_1, x_2, ..., x_n] ∈ ℝ^{n × d}

注意力矩阵 A = softmax(QK^T/√d_k) ∈ ℝ^{n × n}

A[i][j] 表示位置 i 对位置 j 的注意力权重

例如对 "The cat sat on the mat"：

       The  cat  sat  on  the  mat
The   [0.1  0.3  0.2  0.1  0.1  0.2]
cat   [0.2  0.1  0.3  0.1  0.1  0.2]
sat   [0.1  0.2  0.1  0.1  0.1  0.4]  ← sat 关注 mat
on    [0.1  0.1  0.3  0.1  0.1  0.3]
the   [0.1  0.2  0.2  0.1  0.1  0.3]
mat   [0.1  0.1  0.3  0.2  0.1  0.2]
```

### 3.1.5 Self-Attention 的时间和空间复杂度

```
计算复杂度：O(n² · d)
- Q·K^T：n×d_k 乘 d_k×n = O(n²·d_k)
- softmax·V：n×n 乘 n×d_v = O(n²·d_v)

空间复杂度：O(n²)
- 注意力矩阵需要存储 n×n 的矩阵

→ 这就是长序列处理的瓶颈：n=100K 时，n²=10^10
→ 催生了 Flash Attention、Ring Attention 等优化技术
```

## 3.2 Multi-Head Attention

### 3.2.1 动机

**单头注意力的局限**：一个注意力头只能学习一种"关注模式"。但语言理解需要同时关注多种关系：
- 语法关系（主语-谓语-宾语）
- 语义关系（同义词、上下文线索）
- 位置关系（相邻词）
- 指代关系（代词-名词）

### 3.2.2 实现

```
MultiHead(Q, K, V) = Concat(head_1, ..., head_h) · W_O

其中 head_i = Attention(Q · W_Q^i, K · W_K^i, V · W_V^i)

参数：
- h 个注意力头，每个头的维度 d_k = d_model / h
- W_Q^i ∈ ℝ^{d_model × d_k}, W_K^i, W_V^i 同理
- W_O ∈ ℝ^{h·d_v × d_model}
```

**直觉理解**：多头注意力就像一个委员会——每个委员（头）从不同角度分析同一个输入，最后综合所有委员的意见做出决策。

```
                    输入 X
                   ╱  |  ╲
                 ╱    |    ╲
Head 1         Head 2      Head h
(语法关系)    (语义关系)   (位置关系)
  ╲              |           ╱
    ╲            |         ╱
      Concat + Linear Projection
              |
            输出
```

### 3.2.3 参数效率

```
单头注意力参数：3 × d × d = 3d²
多头注意力参数：h × 3 × d × (d/h) + d × d
              = 3d² + d²
              = 4d²

多头注意力只比单头多了一个投影矩阵 W_O 的参数
但获得了 h 种不同的"关注模式"
→ 参数效率极高
```

## 3.3 Positional Encoding

### 3.3.1 为什么需要位置编码？

Self-Attention 是"位置无关"的——打乱输入顺序，注意力权重完全不变。但语言的语义高度依赖词序：
- "狗咬了人" vs "人咬了狗" — 含义完全不同

因此需要将位置信息注入模型。

### 3.3.2 正弦/余弦位置编码

```
PE(pos, 2i) = sin(pos / 10000^(2i/d))
PE(pos, 2i+1) = cos(pos / 10000^(2i/d))

pos：token 在序列中的位置 (0, 1, 2, ...)
i：编码维度的索引
d：模型维度
```

**为什么用正弦/余弦？**

1. **有界性**：值域在 [-1, 1]，不会影响模型稳定性
2. **唯一性**：每个位置有唯一的编码向量
3. **相对位置表达**：任意两个位置的编码差可以通过线性变换得到
   ```
   PE(pos+k) 可以表示为 PE(pos) 的线性函数
   → 模型可以学习关注"前一个词"、"后两个词"等相对位置
   ```
4. **外推能力**：理论上可以处理任意长度的序列（但实际效果有限）

### 3.3.3 RoPE（Rotary Position Embedding）

RoPE 是现代 LLM（LLaMA、Qwen、Mistral）的标配位置编码。

**核心思想**：通过旋转矩阵将位置信息编入 Q 和 K 的内积中。

```
f_q(x_m, m) = (W_q · x_m) · e^{imθ}
f_k(x_n, n) = (W_k · x_n) · e^{inθ}

内积：
<f_q(x_m, m), f_k(x_n, n)> = <W_q·x_m, W_k·x_n> · e^{i(m-n)θ}

→ 注意力分数只依赖于相对位置 (m-n)，而非绝对位置
→ 这正是我们想要的：模型关注的应该是相对位置关系
```

**2D 旋转矩阵实现**：

```
对于 d 维向量，将其两两配对，每对施加旋转：

[q_{2i}  ]     [cos(mθ_i)  -sin(mθ_i)] [q_{2i}  ]
[q_{2i+1}]  =  [sin(mθ_i)   cos(mθ_i)] [q_{2i+1}]

θ_i = 10000^{-2i/d}

这等价于在复数平面上做旋转
```

**RoPE 的优势**：
- 天然编码相对位置（而非绝对位置）
- 注意力分数随距离衰减（高频分量快速旋转导致随机化）
- 可以通过调整基频 θ 来外推到更长序列（NTK-Aware Scaling、YaRN）

**面试重点** 🎯：RoPE 已经成为现代 LLM 的标配，理解其"旋转编码相对位置"的核心思想很重要。

### 3.3.4 ALiBi（Attention with Linear Biases）

```
不修改 Q/K，而是直接在注意力分数上加一个与距离成正比的偏置：

Attention(Q, K, V) = softmax(QK^T/√d_k + m · bias)

bias[i][j] = -|i - j|   （距离越远，惩罚越大）
m：每个头有不同的斜率

→ 自然衰减远距离的注意力
→ 无需位置编码参数，完全在注意力计算中实现
→ 零成本外推到训练时未见过的长度
```

## 3.4 Transformer 为什么能取代 RNN？

### 3.4.1 并行计算

```
RNN：
  h_1 → h_2 → h_3 → ... → h_n  （串行，每步依赖前一步）
  时间复杂度：O(n)，但无法并行
  
Transformer：
  所有位置同时计算注意力（矩阵乘法）
  时间复杂度：O(1)（在 GPU 上并行），但总计算量 O(n²)
  
→ 对于 GPU 来说，n² 次并行计算远快于 n 次串行计算
→ 这是 Transformer 能 scale 的关键原因
```

### 3.4.2 长程依赖

```
RNN 中信息传播：
  位置 1 的信息要传到位置 n → 经过 n-1 步传递
  每步都有信息损失 → 远距离信息严重衰减
  
Transformer 中信息传播：
  位置 1 的信息要传到位置 n → 直接通过注意力连接
  路径长度 = O(1) → 长程依赖建模能力远强于 RNN
```

### 3.4.3 表达能力

```
理论证明（Yun et al., 2020）：
Transformer 是序列到序列函数的万能近似器
→ 给定足够的层数和宽度，可以近似任何序列到序列的映射
```

## 3.5 Transformer 的完整架构

### 3.5.1 编码器块（Encoder Block）

```
输入 x
  ↓
Multi-Head Self-Attention
  ↓
Add & Layer Norm (残差连接 + LN)
  ↓
Feed-Forward Network (FFN)
  ↓
Add & Layer Norm (残差连接 + LN)
  ↓
输出

FFN = Linear(d, 4d) → GELU/SwiGLU → Linear(4d, d)
```

### 3.5.2 解码器块（Decoder Block）

```
输入 x
  ↓
Masked Multi-Head Self-Attention  ← 因果掩码防止看到未来
  ↓
Add & Layer Norm
  ↓
Cross-Attention (Q from decoder, K/V from encoder)
  ↓
Add & Layer Norm
  ↓
Feed-Forward Network
  ↓
Add & Layer Norm
  ↓
输出
```

### 3.5.3 因果掩码（Causal Mask）

```
在自回归生成中，位置 i 只能关注 ≤ i 的位置：

Mask = [1, -∞, -∞, -∞]    位置 0 只能看自己
       [1,  1, -∞, -∞]    位置 1 能看 0 和 1
       [1,  1,  1, -∞]    位置 2 能看 0, 1, 2
       [1,  1,  1,  1]    位置 3 能看所有

应用：softmax(QK^T/√d_k + Mask) · V
-∞ 经过 softmax 后变为 0 → 未来位置的注意力权重为零
```

### 3.5.4 Pre-Norm vs Post-Norm

```
原始 Transformer (Post-Norm):
  x → Attention → x + Attention(x) → LayerNorm → FFN → ...

现代 LLM (Pre-Norm):
  x → LayerNorm → Attention → x + Attention(LN(x)) → ...

Pre-Norm 的优势：
- 梯度传播更稳定（残差路径上没有 LN 阻碍）
- 训练更容易，不需要 warmup
- GPT-2 及之后的模型都采用 Pre-Norm
```

### 3.5.5 KV Cache

```
自回归生成时：
  第 1 步：计算 K_1, V_1
  第 2 步：计算 K_2, V_2，需要 K_1, V_1 来做注意力
  第 n 步：计算 K_n, V_n，需要 K_1...K_{n-1}, V_1...V_{n-1}

→ 如果不缓存，每步都要重算所有之前的 K, V
→ 总计算量：O(n² × L × d)

KV Cache：
  将之前步骤的 K, V 缓存起来
  每步只需计算新 token 的 K, V，然后追加到缓存
  → 总计算量降为 O(n × L × d)

代价：
  显存占用 = 2 × n × L × h × d_k × sizeof(fp16)
  对于 70B 模型、4K 序列 → ~40GB KV Cache
  → PagedAttention (vLLM) 的优化动机
```

### 3.5.6 Flash Attention

```
标准注意力的显存瓶颈：
  需要存储 n × n 的注意力矩阵 → O(n²) 显存

Flash Attention 的核心思想：分块计算
  将 Q, K, V 分块，每次只计算一小块的注意力
  通过在线 softmax 算法避免存储完整注意力矩阵
  → 显存从 O(n²) 降到 O(n)
  → 利用 GPU SRAM（快）减少 HBM（慢）访问
  → 实际速度提升 2-4 倍，显存减少 5-20 倍

技术原理：
  1. 分块：将 Q/K/V 分成小块，每块放入 SRAM
  2. 在线 softmax：不需要完整行的 max 就能计算 softmax
  3. 重计算：反向传播时重算注意力而非存储，以时间换空间
```

## 3.6 架构变体

### 3.6.1 Encoder-Only (BERT)

```
用途：自然语言理解（分类、NER、QA）
特点：双向注意力（每个位置可以看到所有位置）
预训练：Masked Language Model (MLM) — 随机遮挡 15% 的 token，预测被遮挡的 token
```

### 3.6.2 Decoder-Only (GPT)

```
用途：文本生成、代码生成、对话
特点：因果注意力（只能看到当前和之前的位置）
预训练：Causal Language Model — 预测下一个 token
为什么 GPT 路线胜出？
  1. 生成能力天然：Decoder 本身就是生成器
  2. 涌现能力：足够大的 Decoder-Only 模型展现出理解能力
  3. 训练效率：每个 token 都参与训练（BERT 只有 15%）
  4. Scaling 友好：简单架构更容易 scale
```

### 3.6.3 Encoder-Decoder (T5)

```
用途：翻译、摘要、多任务学习
特点：Encoder 理解输入 + Decoder 生成输出
预训练：Span Corruption — 遮挡连续的 token 片段并生成
代表：T5, BART, Flan-T5
```

### 3.6.4 GQA（Grouped Query Attention）

```
MHA (Multi-Head Attention)：每个头有独立的 Q, K, V
MQA (Multi-Query Attention)：所有头共享 K, V
GQA (Grouped Query Attention)：分组共享 K, V

MHA：h 组 Q, h 组 K, h 组 V         → KV Cache 大
MQA：h 组 Q, 1 组 K, 1 组 V         → KV Cache 小但质量下降
GQA：h 组 Q, g 组 K, g 组 V (g < h) → 平衡点

LLaMA-2 70B 使用 GQA (g=8, h=64)
→ KV Cache 减少 8 倍，性能几乎不降
```

### 3.6.5 MoE（Mixture of Experts）

```
标准 FFN：所有 token 经过同一个 FFN
MoE FFN：有 E 个专家 FFN，路由器为每个 token 选择 top-k 个专家

MoE(x) = Σ_{i=1}^{k} G(x)_i · Expert_i(x)
G(x) = TopK(softmax(W_g · x))  （路由器）

优势：
- 总参数量大（所有专家参数之和）→ 高容量
- 激活参数少（每次只用 k 个专家）→ 低计算量
- Mixtral 8x7B：总参数 47B，激活参数 13B

挑战：
- 负载均衡：某些专家可能被过度使用
- 通信开销：分布式训练中专家分布在不同 GPU
- 训练不稳定：路由器训练容易坍缩
```

# 第四章：概率统计与信息论

## 4.1 贝叶斯定理

### 4.1.1 公式与直觉

```
P(A|B) = P(B|A) · P(A) / P(B)

后验 = 似然 × 先验 / 证据
```

**直觉**：贝叶斯定理告诉我们如何根据新证据更新信念。

**经典例子**——医学检测：

```
疾病发病率 P(D) = 0.001（先验）
检测灵敏度 P(+|D) = 0.99（真阳性率）
检测特异度 P(-|¬D) = 0.99（真阴性率）

问：检测阳性，实际患病的概率？

P(D|+) = P(+|D)·P(D) / P(+)
       = P(+|D)·P(D) / [P(+|D)·P(D) + P(+|¬D)·P(¬D)]
       = 0.99 × 0.001 / (0.99 × 0.001 + 0.01 × 0.999)
       = 0.00099 / (0.00099 + 0.00999)
       ≈ 0.09 = 9%

→ 即使检测准确率 99%，阳性结果实际患病概率只有 9%！
→ 因为先验概率（发病率）太低
→ 这就是贝叶斯思维的威力
```

### 4.1.2 在机器学习中的核心地位

```
1. 朴素贝叶斯分类器：
   P(y|x) ∝ P(y) · Π P(x_i|y)

2. 贝叶斯优化：
   用高斯过程作为先验，根据观测更新后验，指导超参数搜索

3. 贝叶斯神经网络：
   对权重施加先验分布，学习后验分布（而非点估计）
   → 提供预测不确定性量化

4. RLHF 中的贝叶斯思想：
   先验 = 预训练模型的行为
   似然 = 人类偏好数据
   后验 = 对齐后的模型
```

## 4.2 最大似然估计 vs 最大后验估计

### 4.2.1 MLE（最大似然估计）

```
θ_MLE = argmax_θ P(D|θ) = argmax_θ Π P(x_i|θ)

等价于：argmax_θ Σ log P(x_i|θ)

直觉：找到使观测数据最"可能"出现的参数
```

**例子——硬币正面概率**：

```
抛硬币 10 次，7 次正面
P(D|p) = C(10,7) · p^7 · (1-p)^3

∂/∂p log P(D|p) = 7/p - 3/(1-p) = 0
→ p_MLE = 7/10 = 0.7
```

### 4.2.2 MAP（最大后验估计）

```
θ_MAP = argmax_θ P(θ|D) = argmax_θ P(D|θ) · P(θ)

等价于：argmax_θ [Σ log P(x_i|θ) + log P(θ)]

→ MLE + 先验 = MAP
```

**MAP 与正则化的等价**：

```
如果先验 P(θ) = N(0, σ²I)（高斯先验）：
  log P(θ) = -||θ||² / (2σ²) + const

MAP = argmax_θ [log P(D|θ) - ||θ||²/(2σ²)]
    = argmin_θ [-log P(D|θ) + (1/2σ²)||θ||²]
    = argmin_θ [NLL + λ||θ||²]

→ 高斯先验 → L2 正则化
→ 拉普拉斯先验 → L1 正则化

面试重点 🎯：L2 正则化等价于高斯先验的 MAP 估计
```

## 4.3 信息论基础

### 4.3.1 熵（Entropy）

```
H(X) = -Σ P(x) · log₂ P(x)

衡量随机变量的"不确定性"或"信息量"
```

**直觉**：
- 公平硬币 → H = 1 bit（完全不确定，需要 1 bit 编码）
- 偏置硬币(P=0.99) → H ≈ 0.08 bit（几乎确定，几乎不需要编码）
- 熵越高 = 越不确定 = 信息量越大

### 4.3.2 交叉熵（Cross-Entropy）

```
H(P, Q) = -Σ P(x) · log Q(x)

用分布 Q 编码真实分布 P 需要的平均比特数
```

**在机器学习中**：
- P = 真实标签分布（one-hot）
- Q = 模型预测的概率分布
- H(P, Q) = 交叉熵损失
- 最小化 H(P, Q) = 让 Q 尽量接近 P

### 4.3.3 KL 散度（Kullback-Leibler Divergence）

```
KL(P || Q) = Σ P(x) · log(P(x) / Q(x))
           = H(P, Q) - H(P)

KL 散度 = 交叉熵 - 熵
```

**性质**：
- KL(P || Q) ≥ 0（Gibbs 不等式）
- KL(P || Q) = 0 当且仅当 P = Q
- 不对称：KL(P || Q) ≠ KL(Q || P)
- 不是距离（不满足三角不等式和对称性）

**在 ML 中的应用**：
1. **VAE 的损失函数**：KL(q(z|x) || p(z))，约束后验接近先验
2. **知识蒸馏**：KL(teacher || student)，学生模拟老师
3. **RLHF 中的 KL 惩罚**：KL(π_RL || π_SFT)，防止 RL 模型偏离 SFT 太远

### 4.3.4 互信息（Mutual Information）

```
I(X; Y) = H(X) - H(X|Y) = H(Y) - H(Y|X) = KL(P(X,Y) || P(X)P(Y))

衡量两个变量之间"共享"了多少信息
```

**直觉**：知道 Y 后，X 的不确定性减少了多少。
- I(X; Y) = 0 → X 和 Y 独立
- I(X; Y) = H(X) → 知道 Y 就完全确定 X

**在 ML 中的应用**：
- 特征选择：选择与标签互信息最大的特征
- 对比学习（SimCLR、CLIP）：最大化正样本对的互信息
- 信息瓶颈理论：深度学习 = 最大化层与标签的互信息 + 最小化层与输入的互信息

## 4.4 概率图模型

### 4.4.1 贝叶斯网络（有向图）

```
用有向无环图（DAG）表示变量之间的因果关系：

    下雨 → 路滑
      ↘       ↘
       堵车 ← 事故

联合概率分解：
P(下雨, 路滑, 堵车, 事故) = P(下雨) · P(路滑|下雨) · P(事故|路滑) · P(堵车|下雨,事故)
```

### 4.4.2 马尔可夫随机场（无向图）

```
用无向图表示变量之间的关联关系：

  X1 — X2 — X3
  |          |
  X4 — X5 — X6

联合概率通过势函数定义：
P(X) = (1/Z) · Π_c ψ_c(X_c)

Z 是归一化常数（配分函数）
ψ_c 是团势函数
```

### 5.4.1 Herbert Simon 的有限理性

```

- 信息完全
- 计算能力无限
- 做出最优决策

有限理性（现实）：

→ AI Agent 天然是"有限理性"的：
  - 上下文窗口有限（信息不完全）
  - 推理步骤有限（计算资源有限）

```

```
期望效用最大化：
  argmax_a Σ P(s|a) · U(s)

其中 a 是动作，s 是结果状态，U 是效用函数

→ Agent 决策的理论基础
→ 奖励函数设计 = 效用函数定义

应用：

# 附录 A：关键数学推导详解

## A.1 Softmax 函数的性质与数值稳定性

### A.1.1 定义与概率解释

```
softmax(z_i) = exp(z_i) / Σ_j exp(z_j)

性质：
1. 输出 ∈ (0, 1)
2. 所有输出之和 = 1（概率分布）
3. 保序性：z_i > z_j → softmax(z_i) > softmax(z_j)
4. 温度参数：softmax(z_i / T)
   - T → 0：趋向 argmax（确定性选择）
   - T → ∞：趋向均匀分布（随机选择）
   - T = 1：标准 softmax
```

### A.1.2 数值稳定性

```
问题：exp(z_i) 在 z_i 很大时会溢出

解决：利用 softmax 的平移不变性
softmax(z_i) = softmax(z_i - max(z))

exp(z_i - max(z)) ≤ exp(0) = 1  → 不会溢出

实现：
def stable_softmax(z):
    z_shifted = z - np.max(z)
    exp_z = np.exp(z_shifted)
    return exp_z / np.sum(exp_z)
```

### A.1.3 温度在 LLM 中的应用

```
LLM 生成时：
  P(next_token = w_i) = softmax(logit_i / T)

temperature = 0.0：贪心解码（总是选最可能的 token）→ 确定性、重复
temperature = 0.7：平衡创造性和准确性 → 对话、写作的常用值
temperature = 1.0：按原始概率采样 → 最"忠实"于模型分布
temperature = 1.5+：高度随机 → 创意写作、头脑风暴

面试重点 🎯：temperature 不改变 logits 的排序，只改变概率分布的"尖锐程度"
```

## A.2 梯度下降的收敛性分析

### A.2.1 凸优化中的收敛速率

```
对于 L-Lipschitz 光滑的凸函数，学习率 η = 1/L 时：

GD 收敛速率：f(x_t) - f(x*) ≤ O(1/t)
SGD 收敛速率：E[f(x_t)] - f(x*) ≤ O(1/√t)

→ SGD 收敛更慢，但每步计算量远小于 GD
→ 总体计算效率 SGD 更高
```

### A.2.2 非凸优化（深度学习的实际情况）

```
深度学习的损失函数是非凸的：
- 存在大量鞍点（高维空间中鞍点远多于局部最小值）
- 局部最小值之间的损失差异通常很小
- SGD 的噪声反而有助于逃离鞍点

关键洞察（2014, Dauphin et al.）：
  在 n 维空间中，随机鞍点的概率远高于局部最小值
  因为鞍点只需要一个方向是上凸的即可
  而局部最小值需要所有方向都是下凸的
  → P(鞍点) >> P(局部最小值)

→ 深度学习的优化困难主要来自鞍点而非局部最小值
→ SGD 的随机性 + 动量有助于逃离鞍点
```

### A.2.3 学习率调度策略

```
1. Step Decay：每隔 N 个 epoch 将学习率乘以 γ
   η_t = η_0 · γ^(t/N)

2. Cosine Annealing：
   η_t = η_min + (η_max - η_min) · (1 + cos(πt/T)) / 2
   
   平滑地从最大值衰减到最小值，然后重启

3. Warmup + Decay（LLM 标配）：
   前 warmup_steps 步：η_t = η_max · t / warmup_steps
   之后：cosine 或线性衰减

   为什么需要 warmup？
   - 初始时参数远离最优，梯度方向不准确
   - 大学习率可能导致参数跳到很差的区域
   - 小学习率起步 → 稳定方向 → 再加大探索

4. 一些现代调度器：
   - WSD (Warmup-Stable-Decay)：用于大模型预训练
   - OneCycle：先升后降，训练更快
```

## A.3 激活函数深度对比

```
函数名    | 公式                  | 导数范围    | 优点            | 缺点
----------|-----------------------|------------|-----------------|----------------
Sigmoid   | 1/(1+e^(-x))         | (0, 0.25]  | 概率输出        | 梯度消失、非零均值
Tanh      | (e^x-e^(-x))/(...)   | (0, 1]     | 零均值          | 梯度消失
ReLU      | max(0, x)            | {0, 1}     | 简单高效        | 死神经元问题
Leaky ReLU| max(0.01x, x)        | {0.01, 1}  | 解决死神经元    | 多一个超参数
GELU      | x·Φ(x)              | ~[0, 1.05] | 平滑、BERT用    | 计算稍慢
SiLU/Swish| x·σ(x)              | ~[-0.28,∞) | 平滑、性能好    | 计算稍慢
SwiGLU    | SiLU(xW₁)⊙(xW₂)    | -          | LLM最佳         | 参数多50%
```

**SwiGLU 为什么成为 LLM 标配？**

```
标准 FFN：FFN(x) = GELU(xW₁)W₂
SwiGLU FFN：FFN(x) = [SiLU(xW₁) ⊙ (xW₃)]W₂

SwiGLU 的门控机制：
- xW₁ 经过 SiLU 产生"激活信号"
- xW₃ 产生"内容信号"
- 两者逐元素相乘 = 门控
- 类似 LSTM 的门控思想

虽然参数多了 50%（3 个矩阵 vs 2 个），
但在相同参数量下性能更好（通过减小隐藏层维度来补偿）
→ Google PaLM、Meta LLaMA、Mistral 等都采用
```

## A.4 初始化策略

```
参数初始化的目标：让每一层的输出方差保持稳定

1. Xavier/Glorot 初始化（适用于 Sigmoid/Tanh）：
   W ~ U(-√(6/(n_in + n_out)), √(6/(n_in + n_out)))
   或 W ~ N(0, 2/(n_in + n_out))
   
   原理：让前向传播和反向传播中方差都不变

2. He/Kaiming 初始化（适用于 ReLU）：
   W ~ N(0, 2/n_in)
   
   原理：ReLU 将一半激活值置零，需要补偿方差

3. 现代 LLM 的初始化：
   - 通常用 N(0, 0.02) 或类似小方差高斯
   - 残差层的输出有时额外缩放 1/√N_layers
   - GPT-2 对残差连接的最后一个线性层缩放 1/√(2·N_layers)
```

# 附录 G：深度学习中的数学工具箱

## G.1 线性代数核心

### G.1.1 矩阵分解

**特征值分解（Eigendecomposition）**：

```
A = QΛQ^(-1)

其中：
- Q 的列向量是特征向量
- Λ 是对角矩阵，对角元素是特征值

应用：
- PCA 的数学基础
- 理解神经网络的 Hessian 矩阵（二阶优化）
- 分析 RNN 梯度传播（特征值 > 1 → 爆炸，< 1 → 消失）
```

**奇异值分解（SVD）**：

```
A = UΣV^T

其中：
- U：左奇异向量矩阵
- Σ：对角矩阵，对角元素是奇异值（非负，降序排列）
- V^T：右奇异向量矩阵

应用：
- 低秩近似（LoRA 的数学基础）
- 数据压缩
- 潜在语义分析（LSA）
```

**LoRA 与 SVD 的联系**：

```
LoRA 的核心假设：微调时权重变化 ΔW 是低秩的

ΔW ≈ BA，其中 B ∈ ℝ^(d×r), A ∈ ℝ^(r×d), r << d

这等价于 SVD 的前 r 个奇异值近似
→ 参数量从 d² 降到 2dr
→ 对于 d=4096, r=16：参数降低 128 倍

为什么低秩假设成立？
- 微调通常只是微调模型行为，不需要大幅改变权重
- 实验表明，微调后的权重变化确实是低秩的
- rank=8~64 通常就足够了
```

### G.1.2 矩阵求导

```
深度学习中常用的矩阵求导公式：

1. ∂(Wx)/∂x = W^T
2. ∂(x^T W x)/∂x = (W + W^T)x
3. ∂(trace(AB))/∂A = B^T
4. ∂||x||²/∂x = 2x
5. ∂(softmax(x))/∂x = diag(p) - pp^T  （其中 p = softmax(x)）

链式法则的矩阵版本（Jacobian）：
如果 z = f(y), y = g(x)
则 ∂z/∂x = (∂z/∂y)(∂y/∂x) = J_f · J_g
```

### G.1.3 张量操作

```
深度学习中的核心张量操作：

操作          | 数学含义          | PyTorch 实现
-------------|-------------------|------------------
矩阵乘法      | C = AB            | torch.matmul(A, B)
逐元素乘法     | C = A ⊙ B         | A * B
外积          | C = a ⊗ b         | torch.outer(a, b)
转置          | A^T               | A.T / A.transpose()
广播          | 自动扩展维度       | 自动支持
```

### G.2.1 常用概率分布

```
分布          | 参数      | 用途
-------------|----------|----------------------------------
伯努利        | p        | 二分类标签

Dirichlet     | α_1..α_k | 主题模型 / 浓度参数
```

### G.2.2 重参数化技巧（Reparameterization Trick）

```
问题：VAE 中需要对潜在变量 z ~ q(z|x) 求梯度
但从分布中采样这个操作不可微

解决：重参数化
z = μ + σ ⊙ ε,  ε ~ N(0, I)

→ 将随机性从 z 转移到 ε
→ μ 和 σ 是确定性函数，可以正常求梯度
→ 这个技巧使得 VAE 可以端到端训练
```

### G.2.3 变分推断（Variational Inference）

```
目标：计算后验分布 P(z|x)
问题：直接计算不可行（需要计算边际似然 P(x) = ∫ P(x|z)P(z)dz）

解决：用一个简单分布 q(z|x) 近似 P(z|x)
目标：min KL(q(z|x) || P(z|x))

等价于最大化 ELBO（Evidence Lower BOund）：
ELBO = E_q[log P(x|z)] - KL(q(z|x) || P(z))
     = 重构损失 + KL 正则化
```

## G.3 最优化理论

### G.3.1 凸优化 vs 非凸优化

```
凸函数：f(λx + (1-λ)y) ≤ λf(x) + (1-λ)f(y)
→ 局部最小值 = 全局最小值
→ 梯度下降保证收敛到全局最优

非凸函数（深度学习的现实）：
→ 存在多个局部最小值和鞍点
→ 梯度下降只保证收敛到驻点
→ 但实践中效果很好
```

### G.3.2 二阶优化

```
Newton 方法：x_{t+1} = x_t - H^(-1) · g
其中 H = ∂²L/∂x² 是 Hessian 矩阵

优势：收敛速度更快（二次收敛 vs 线性收敛）
劣势：Hessian 矩阵计算和存储代价太高 O(n²)
→ 对于 LLM 的数十亿参数不可行

近似方法：
- L-BFGS：用有限内存近似 Hessian 的逆
- Natural Gradient：用 Fisher 信息矩阵代替 Hessian
- Shampoo / SOAP：分块近似的二阶优化器
- K-FAC：用 Kronecker 分解近似 Fisher 矩阵

实践中 Adam 可以被视为对角线近似的二阶方法
```

# 附录 I：从基础理论到 LLM 的桥梁

## I.1 预训练的信息论视角

```
LLM 预训练目标：最小化 Next Token Prediction 的交叉熵

L = -E[log P(x_t | x_1, ..., x_{t-1})]

从信息论看：
- 等价于最小化模型分布 Q 和真实分布 P 的交叉熵 H(P, Q)
- 最小化交叉熵 = 最小化 KL(P || Q) + H(P)
- 因为 H(P) 是常数，等价于最小化 KL 散度
- 即让模型分布尽量接近真实数据分布

困惑度（Perplexity）：
PPL = exp(L) = 2^H(P,Q)
- PPL = 10 → 模型在每个位置平均有 10 个等可能的选择
- PPL 越低 → 语言建模越好
```

## I.2 涌现能力的理论解释

```
涌现能力（Emergent Abilities）：
模型规模超过某个阈值后，突然出现的新能力

可能的解释：
1. 相变理论：类似物理中的相变，量变到质变
2. 组合效应：多个子技能在足够大的模型中组合
3. 度量效应：能力一直在缓慢增长，但评测指标是阶跃的
4. 记忆容量：更大模型能记住更多模式和规则
```

## I.3 In-Context Learning 的理论

```
ICL（In-Context Learning）：
不更新模型参数，仅通过在 prompt 中提供示例来学习新任务

理论解释：
1. 隐式贝叶斯推断：LLM 在预训练中学习了任务先验
2. 内部优化器：Transformer 在前向传播中实现了梯度下降
3. Induction Heads：两层注意力头组合实现模式匹配
```

## I.4 Scaling Laws 的数学表述

```
Kaplan et al. (2020) 发现的幂律关系：
L(N) = (N_c / N)^α_N    性能 ∝ 参数量的幂
L(D) = (D_c / D)^α_D    性能 ∝ 数据量的幂
L(C) = (C_c / C)^α_C    性能 ∝ 计算量的幂

Chinchilla 定律：
给定计算预算 C，最优配置是 N 和 D 同比例增长
→ LLaMA 按此原则训练，效果显著优于盲目扩大模型
```

| DeepSpeed ZeRO-1 | ~25% | 略慢 | 中 |
| DeepSpeed ZeRO-2 | ~50% | 慢一些 | 中 |
| DeepSpeed ZeRO-3 | ~75% | 更慢 | 高 |
| 模型并行(TP) | 按 GPU 数分 | 通信开销 | 高 |
| 流水线并行(PP) | 按 GPU 数分 | 流水线气泡 | 高 |

### 显存估算（7B 参数模型）

```
模型参数(fp16)：7B x 2 bytes = 14 GB
优化器状态(fp32)：7B x 8 bytes = 56 GB（Adam: m, v 各 4 bytes）
梯度(fp16)：7B x 2 bytes = 14 GB
激活值：取决于 batch size 和序列长度
总计 ~84 GB + 激活值 -> 需要多卡训练
```

## L.4 分布式训练策略

```
数据并行（Data Parallelism）：
- 每个 GPU 持有完整模型副本
- 不同 GPU 处理不同数据
- 梯度同步：AllReduce
- 适用：模型能放进单卡

FSDP（Fully Sharded Data Parallel）：
- 模型参数、梯度、优化器状态都分片到不同 GPU
- 计算时按需聚合
- PyTorch 原生支持

张量并行（Tensor Parallelism）：
- 将单个层的参数分片到多 GPU
- 需要节点内高速互联（NVLink）

流水线并行（Pipeline Parallelism）：
- 不同层放在不同 GPU
- 数据按 micro-batch 流水线处理

实际大模型训练通常是三者的组合（3D 并行）
```

# 附录 M：Transformer 内部机制

## M.1 Induction Heads

```
Induction Head 是 Transformer 中最重要的内部机制之一

功能：实现 [A][B]...[A] -> [B] 的模式匹配

由两个注意力头组成：
1. Previous Token Head：关注前一个 token
2. Induction Head：寻找与当前 token 匹配的 key

这是 In-Context Learning 的核心机制
实验发现在训练的特定阶段突然出现（相变）
```

## M.2 Superposition

```
超叠态（Superposition）：
神经网络用 d 维空间表示远多于 d 个概念

- d 维空间只有 d 个正交方向
- 但可以有 d^2 量级的近似正交方向
- Sparse AutoEncoder 可以将叠加的特征分解出来
```

# 附录 N：量化知识速查

## N.1 模型大小估算公式

```
每个 Transformer 层的参数：
- Self-Attention: 4 x d^2 (Q, K, V, O)
- FFN (SwiGLU): 3 x d x d_ff
- LayerNorm: 2 x d

总参数 = L x (4d^2 + 3d*d_ff + 2d) + V*d

LLaMA-7B: L=32, d=4096, d_ff=11008, V=32000
每层 = 4*4096^2 + 3*4096*11008 = ~202M
总计 = 32*202M + 32000*4096 = ~6.7B
```

## N.2 计算量估算

```
前向传播 FLOPs = 2 x 参数量 x 序列长度
训练 FLOPs = 6 x 参数量 x 数据量(tokens)

示例（训练 LLaMA-7B on 1T tokens）：
FLOPs = 6 x 7B x 1T = 4.2 x 10^22
= ~75,000 A100 GPU-hours（50% 利用率）
= ~100 天 x 32 张 A100
```

## N.3 推理延迟估算

```
7B 模型 + A100 80GB：
Memory bandwidth: 2 TB/s
每个 token 读取 14 GB (fp16)
延迟 = 14 GB / 2 TB/s = 7 ms/token
= ~143 tokens/sec（理论上限）
实际 50-100 tokens/sec
```
